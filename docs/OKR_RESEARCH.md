# OKR Research Summary

**Purpose:** Ground Offee’s product and language in how OKRs actually work in practice.

---

## 1. What is OKR?

**OKR = Objectives and Key Results**

- **Objective** — Qualitative, ambitious outcome: *what* we want to achieve (e.g. “Become the default choice for skill credentials in India”).
- **Key Results (KRs)** — 3–5 measurable results that prove the objective is being achieved: *how* we’ll know we got there (e.g. “Issue 10L Skill Passports,” “65 issuers live”).

**Core idea:** OKRs set **direction and ambition**; they are not task lists. Key results should describe **outcomes**, not activities.

---

## 2. OKR vs KPI

| | OKR | KPI |
|---|-----|-----|
| **Role** | Set direction; drive change | Measure ongoing performance |
| **Question** | “Where do we want to be?” | “How are we doing right now?” |
| **Typical horizon** | Quarter / 6 months / year | Ongoing (weekly/monthly) |
| **Stretch** | Often ambitious (70% = success) | Often “health” metrics (e.g. uptime, NPS) |

**In practice:** Use both. OKRs define the goal; KPIs (and daily/weekly numbers) show progress. Offee’s “Daily Leader Scorecard” (output + pipeline + quality KPIs) is the **operational layer** that feeds into **Key Result** progress (weekly scorecard).

---

## 3. DRI (Directly Responsible Individual)

- **One owner per Key Result** — No “we.” One named person is **accountable** for each KR.
- **DRI ≠ doer of all work** — DRI **shepherds**: coordinates, unblocks, tracks, reports. Others can do the work.
- **Why it matters** — Shared ownership → “someone else will do it.” Single DRI → clear accountability.
- **Best practice** — One DRI per KR (or at most two in rare, truly shared cases).

**Offee alignment:** Your model (each KR has one DRI, leaders have max 3 KRAs) matches this. Weekly Scorecard = DRI reporting progress on their KRs.

---

## 4. Cadence: How Often to Update

| Cadence | Use case | Research insight |
|---------|----------|------------------|
| **Quarterly (or 6M)** | Set/refresh OKRs and KRs | Standard. Some use annual + quarterly (dual cadence). |
| **Weekly** | Update KR progress, blockers, next steps | **Strongest predictor of OKR success.** Teams that update every 7 days are 40%+ more likely to hit goals. 20–30 min check-ins: wins, next week focus, blockers, RAG status, KR progress. |
| **Daily** | Operational metrics, blockers, decisions | Short (10–15 min). Focus on **progress and blockers**, not task lists. |

**Offee alignment:**  
- **Daily Leader Scorecard** = daily operational pulse (output/pipeline/quality + blockers).  
- **Weekly KR Scorecard** = weekly KR update (target vs actual vs next week).  
This matches “weekly cadence for OKR success” + “daily rhythm for execution.”

---

## 5. Common Mistakes (What to Avoid)

1. **OKRs as task lists** — Objectives and KRs should be outcomes, not “do 10 meetings.”
2. **No link to strategy** — OKRs should tie to company vision and priorities.
3. **Too many OKRs** — Fewer, focused objectives beat many. 2–4 key results per objective is typical.
4. **No regular updates** — Sporadic updates → late surfacing of risk. Weekly (at least) is key.
5. **Vague or KPI‑only KRs** — Key results should be measurable and tied to the objective, not just existing KPIs renamed.
6. **No single owner** — Unclear ownership → diffusion of responsibility and missed KRs.

---

## 6. How OKR Software Typically Works

From research on OKR tools and execution:

- **Set** — Define company/team OKRs and KRs; assign DRIs; set quarter/period.
- **Align** — Make OKRs visible; show how team/individual work connects (alignment map).
- **Track** — Dashboards: on track / at risk / behind; progress %; real-time view.
- **Check-in** — Weekly (or biweekly) updates: confidence, KR progress, blockers, next steps.
- **Review** — End of quarter: results, learnings, input to next cycle.
- **Accountability** — Role-based: leaders own objectives; DRIs own key results; software enforces “one owner per KR.”

**Offee in this frame:**  
- **Set** → OKRs + Key Results + DRI (admin/CEO).  
- **Track** → CEO Dashboard (company view); My Dashboard (my KRs, my decisions).  
- **Check-in** → Daily Scorecard (leader KPIs + blockers); Weekly Scorecard (KR target/actual/next week).  
- **Decisions** → Log and assign owner/due; escalate overdue → execution discipline.

---

## 7. One-Line Summary

**OKR** = set a few ambitious **objectives**, each with 3–5 measurable **key results**, each owned by **one DRI**, updated **weekly** (and supported by a **daily** operational pulse); avoid task lists and unclear ownership.
